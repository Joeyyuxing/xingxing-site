# 🚀 星星的小破站 — 部署指南

## 总览

你的网站由两部分组成：
- **前端页面**：部署到 GitHub Pages（免费、全球 CDN）
- **后端存储**：腾讯云开发 CloudBase（免费额度、国内加速）

完成后效果：你在网站发的帖子/图片，所有访客都能看到。

---

## 第一步：配置腾讯云开发 CloudBase（5 分钟）

### 1.1 注册/登录腾讯云

打开 https://cloud.tencent.com/ ，用微信扫码登录即可（不需要绑定域名备案）。

### 1.2 创建云开发环境

1. 进入 [云开发控制台](https://console.cloud.tencent.com/tcb)
2. 点击「新建环境」
3. 选择**免费体验版**（每月 3000 资源点，个人网站完全够用）
4. 地域选**上海**（国内访问最快）
5. 记下你的**环境 ID**（类似 `xingxing-xxxxx`）

### 1.3 开启匿名登录 / 获取 Publishable Key

1. 在云开发控制台，进入你的环境
2. 左侧菜单选**环境配置** → **API Key 配置**
3. 点击生成 **Publishable Key**（这个 key 可以安全地放在前端代码中）
4. 记下这个 Key

### 1.4 配置安全域名

1. 左侧菜单选**环境配置** → **安全配置**
2. 添加你的 GitHub Pages 域名，例如：
   - `joeyyuxing.github.io`（你的 GitHub Pages 域名）
   - `localhost`（本地调试用）
3. 等待约 10 分钟生效

### 1.5 创建数据库集合

1. 左侧菜单选**数据库**
2. 点击「新建集合」，名称填 `posts`
3. 点击集合 → **权限设置**：
   - 选择 **所有用户可读，仅管理员可写**
   - 或选择 **所有用户可读可写**（更简单，适合个人站）

### 1.6 填入配置

打开 `app.js`，找到顶部的配置区域（约第 598-599 行），替换：

```javascript
const TCB_ENV_ID = 'xingxing-xxxxx';        // ← 你的环境 ID
const TCB_ACCESS_KEY = 'xxxxxxxxx';          // ← 你的 Publishable Key
```

## 第二步：部署到 GitHub Pages（5 分钟）

### 2.1 创建 GitHub 仓库

1. 打开 https://github.com/new（需要 VPN）
2. 仓库名填 `xingxing-site`（或任意名称）
3. 选择 **Public**（GitHub Pages 免费版需要公开仓库）
4. 点击 Create repository

### 2.2 上传文件

方法一：网页直接上传（最简单）
1. 在新建的仓库页面，点击 **uploading an existing file**
2. 把 `xingxing-site` 文件夹里的 `index.html` 和 `app.js` 拖进去
3. 点击 Commit changes

方法二：用 Git 命令行
```bash
cd xingxing-site
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的用户名/xingxing-site.git
git push -u origin main
```

### 2.3 启用 GitHub Pages

1. 进入仓库 → Settings → Pages
2. Source 选择 **Deploy from a branch**
3. Branch 选择 `main`，目录选 `/ (root)`
4. 点击 Save
5. 等待 1-2 分钟，你的网站就上线了！

访问地址：`https://你的用户名.github.io/xingxing-site/`

### 2.4 记得在 CloudBase 添加安全域名

把上面得到的 GitHub Pages 域名（如 `joeyyuxing.github.io`）添加到 CloudBase 的安全域名列表中（第一步 1.4 中操作）。

## 第三步：验证

1. 打开你的 GitHub Pages 链接
2. 进入飞船 → 用管理员模式发一条帖子
3. 用另一个浏览器/手机打开同一个链接
4. 确认能看到刚才发的帖子 ✅

## 常见问题

### Q: GitHub Pages 国内能打开吗？
A: 大部分情况可以，但偶尔可能较慢。如果你在意速度，可以同时保留 Netlify 部署作为备份（把同样的文件拖到 https://app.netlify.com/drop）。

### Q: 免费额度够用吗？
A: CloudBase 免费体验版每月 3000 资源点，对于个人小站（几十篇帖子、日访问几百次）完全够用。

### Q: 如何更新网站？
A: 修改代码后，重新上传到 GitHub 仓库即可（push 或网页上传覆盖），GitHub Pages 会自动重新部署。

### Q: 数据库权限怎么设？
A: 推荐「所有用户可读可写」，因为你的管理员验证在前端密码层。如果想更安全，可以设「所有用户可读，仅管理员可写」，但需要额外配置云函数处理写入。
